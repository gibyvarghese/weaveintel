Core Concepts

  • Model: Unified interface for LLMs (OpenAI, Anthropic, Google, Ollama, llama.cpp, etc.).
  • Tool: Typed, governed function with risk, policy, and audit.
  • Agent: LLM-driven loop (one-shot or long-running) that orchestrates tool use.
  • Mesh: Group of live agents sharing state, schedule, and event bus.
  • Policy/Contract: Declarative rules for tool use, approval, and evidence.

Monorepo Package Map

  • @weaveintel/core: Types, contracts, and context.
  • @weaveintel/agents: One-shot and chat-style ReAct agents.
  • @weaveintel/live-agents: Long-running, scheduled, multi-agent mesh runtime.
  • @weaveintel/tools: Tool registry, risk, policy, and audit pipeline.
  • @weaveintel/prompts: Prompt templates, fragments, frameworks, and versioning.
  • @weaveintel/evals: Evaluation datasets, rubrics, and scoring.
  • @weaveintel/workflows: Composable, DB-driven workflow engine.
  • @weaveintel/triggers: Event-driven triggers and dispatch.
  • @weaveintel/contracts: Evidence ledger and reproducibility.
  • @weaveintel/observability: Tracing, cost, and audit.
  • @weaveintel/replay: Deterministic replay and idempotency.
  • @weaveintel/skills: Behavior packs and skill bundles.
  • @weaveintel/a2a: Multi-agent communication bus.

geneWeave Reference App

The geneWeave app is a full-stack chat and admin dashboard built entirely from @weaveintel/* packages. It demonstrates:

  • Chat engine with tool calling, policy, and audit
  • Admin dashboard for tools, skills, policies, workflows, triggers, and more
  • Scientific validation pipeline (multi-agent, evidence, verdicts)
  • Live-agent mesh orchestration and observability
  • DB-driven configuration and extensibility
