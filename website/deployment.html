Deployment & Development

Home Architecture geneWeave

Deployment Options

  • Docker Compose: docker-compose.yml for local or single-node deployment
  • Kubernetes: deploy/kubernetes.yaml for scalable, production-grade clusters
  • Azure Container App: deploy/azure-container-app.yaml
  • AWS ECS & AppRunner: deploy/aws-ecs-task.json, deploy/aws-apprunner.yaml
  • GCP Cloud Run: deploy/gcp-cloudrun.yaml
  • DigitalOcean App Platform: deploy/digitalocean-app.yaml
  • Vercel, Railway, Fly.io: Supported via vercel.json, railway.toml, fly.toml

Development Workflow

  • Monorepo managed with Turborepo and TypeScript
  • All packages are ESM and composable
  • Run npm run build to build all packages
  • Run npx tsx examples/XX-example.ts to execute any example
  • Tests: npx vitest run for all packages and apps
  • Extensible via packages/ and apps/ directories

Configuration

  • Provider keys via environment variables (e.g., OPENAI_API_KEY, ANTHROPIC_API_KEY)
  • App config via .env and app.json
  • Database: SQLite by default (./geneweave.db), Postgres/Redis/Mongo/Dynamo supported for live-agents

Contributing

  • Follow the code style (.prettierrc) and commit guidelines
  • Open a pull request with a clear description
  • See README.md and CONTRIBUTING.md for more
