What are Live-Agents?

Live-agents are long-running, stateful agents that operate on a schedule, accumulate learnings, and coordinate across a mesh. They are ideal for research, automation, compliance, and event-driven workflows.

Key Use Cases

  1. Long-lived research assistant: Iterative analysis, persistent findings, and context retention. (examples/52-live-agents-basic.ts)
  2. Promotional workflow automation: Multi-channel monitoring, action triggers, and optimization. (examples/53-live-agents-cross-mesh-account-binding.ts)
  3. Cross-mesh collaboration and handoff: Evidence contracts, bridges, and async review. (examples/54-live-agents-cross-mesh-bridges.ts)
  4. Compression for long-running agents: Daily summaries, token budgeting, and scalable memory. (examples/55-live-agents-compression.ts)
  5. Account binding and credential scoping: Human-controlled permissions and runtime validation. (examples/56-live-agents-account-binding-guardrail.ts)
  6. Mesh administration and telemetry: Real-time status, heartbeat, and observability. (examples/57-live-agents-mesh-administration.ts)

Core Patterns

  • StateStore: Persists agent state across restarts and failures.
  • Heartbeat: Schedules and resumes work automatically.
  • Account Binding: Human-controlled, fine-grained capability grants.
  • Bridge: Cross-mesh messaging and contract handoff.
  • CompressionStrategy: Summarizes and prunes long histories for scalable inference.
  • Mesh API: HTTP endpoints for mesh, agent, and heartbeat status.

Minimal Live-Agent Example

import { weaveLiveAgent, weaveInMemoryStateStore } from '@weaveintel/live-agents';

const store = weaveInMemoryStateStore();
const agent = weaveLiveAgent({
  name: 'analyst',
  model: myModel,
  tools: myTools,
  systemPrompt: 'Research and summarize findings.'
});

await agent.runTick({ inbound: { body: 'Start research' } }, store);

See examples/52-live-agents-basic.ts for a full runnable demo.
