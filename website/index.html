What is weaveIntel?

weaveIntel is a TypeScript framework for building production-grade AI applications. It provides composable, vendor-neutral building blocks for everything from a single chat completion to a fleet of long-running multi-agent meshes — with tool calling, governance, observability, and resilience built in.

Quick Start

  1. Clone the repo: git clone https://github.com/gibyvarghese/weaveintel.git
  2. Install dependencies: npm install
  3. Set your provider key(s): export OPENAI_API_KEY=sk-...
  4. Start the reference app: ./scripts/start-geneweave.sh
  5. Open http://localhost:3500 and sign up.

Core Concepts

  • Model: Adapter to an LLM (chat, stream, embed, vision, structured output).
  • Tool: Typed function an agent can call, with risk class, policy, and audit.
  • Agent: LLM-driven loop that picks tools to satisfy a goal. weaveAgent (one-shot) and weaveLiveAgent (long-running).
  • Mesh: Group of live agents sharing state, schedule, and event bus.
  • Policy / Contract: Declarative rules for tool use, approval, and evidence.

Capability Walkthroughs

  • Talk to a model
  • Add tools (weaveAgent)
  • Long-running agents (weaveLiveAgent)
  • Connect to external systems (MCP)
  • Multi-agent communication (A2A)
  • Governance: policy, approval, audit
  • Resilience: rate limits, circuit breakers, retries
  • Memory, RAG, and knowledge graphs
  • Observability and replay
  • Evaluation

Live-Agents Use Cases

  • Long-lived research assistant
  • Promotional workflow automation
  • Cross-mesh collaboration and handoff
  • Compression for long-running agents
  • Account binding and credential scoping
  • Mesh administration and telemetry

The geneWeave Reference App

geneWeave is a full-stack chat app and admin dashboard built only from @weaveintel/* packages. It demonstrates all core and advanced capabilities in a real-world, extensible application.

Deployment & Development

  • Deploy with Docker, Kubernetes, Azure, AWS, GCP, or DigitalOcean using manifests in deploy/.
  • Monorepo managed with Turborepo, TypeScript, and ESM.
  • Extensible via modular packages in packages/.
